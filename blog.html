Q: Discuss the scope of var, let, and const

Ans:
var:
var is a JavaScript keyword used to declare variables. Declaring something with var can be used both outside and inside the function and its value can be changed.

let:
let is also a JavaScript keyword. The variable scope declared by let is block scope. That is, the variable declared by let can be used only within the block in which it is declared and its value can be changed.

const:
The scope of a variable declared by const is block scope and cannot be changed. That is, the value of a variable declared by const once set cannot be changed later.

Q: Tell us the use cases of null and undefined

Ans: The main difference between null and undefined in JavaScript is that null is a value that represents the absence of a value, while undefined is a keyword that represents a variable that has not been assigned a value.

Q: What do you mean by REST API?

Ans: REST API stands for Representational State Transfer Application Programming Interface. It is an architectural style for designing APIs (Application Programming Interfaces) that are loosely coupled and easy to use. REST APIs are based on the HTTP protocol and use standard HTTP verbs (GET, POST, PUT, DELETE) to perform operations on resources. REST APIs are often used to connect different applications and systems, and to allow users to interact with data over the internet. They are a popular choice for building APIs because they are easy to understand and implement, and they are widely supported by different programming languages and platforms.
